Cheat Sheet

The whole app in one page. Skim it before an interview.

1 The one big idea

Data flows one direction. Components are handed their data; they never fetch their own.

lib/
the brain — data & rules
a page fetches data
app/
pages / URLs — assemble + fetch
passes data down as props
components/
UI pieces — props in → JSX out
Data flows down (props)  ·  events flow up (callback functions like onOpen, onChange)

4 Server vs Client Components

The #1 interview topic in modern Next.js.

Default

Server Component

  • Runs on the server
  • Can await data directly
  • Ships zero JS (fast)
  • No state / no onClick
  • Can't be interactive
"use client"

Client Component

  • Runs in the browser
  • useState, onClick, effects
  • Interactive
  • Can't be async / fetch DB
  • Ships JS (the cost)

Fetch high on the server, go interactive low on the client.   page.tsx fetches → Wall (client) handles filters + modal.

7 Say these confidently

Component — a function that returns JSX (UI).
Props — inputs passed to a component. Read-only. Data flows down.
State (useState) — data that changes over time; changing it re-renders the UI.
Callback prop — a function passed down so a child can notify the parent (events flow up).
Conditional render — {loved && <Badge/>} shows only if loved.
List render — {items.map(x => <Row key={x.id}/>)}; key is required.
useEffect — run side-effects after render (key listeners, scroll lock).
useMemo — cache derived data; recompute only when dependencies change.
Context — share global-ish state (theme) without prop-drilling.
Custom hook — reusable logic starting with use (useFilteredCafes).
Union type — "him" | "her" | "shared" — one of a fixed set.
Omit<Cafe,"id"> — that type minus some fields.

8 The stack, in one breath

If they say "tell me about this project" "It's a Next.js App Router app in React + TypeScript, styled with Tailwind wired to CSS-variable design tokens, with Supabase for the database, photo storage, and auth. I fetch on the server and pass data to client components for interactivity. Business rules and the data shape each live in one place, so changes are localized and the type system catches anything I miss."
